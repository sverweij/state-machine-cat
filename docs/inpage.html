Embedding state-machine-cat scripts in html

In your head: 

<script 
  src="https://state-machine-cat.js.org/state-machine-cat-inpage.min.js"
  defer>
</script>

Anywhere in your body:

<script type="text/x-smcat">
  on [color="darkgreen" active],
  off [color="maroon"];

  on => off [color="red"]: flickSwitch()
    / makeNoise("off.wav");
  off => on [color="#009900"]: flickSwitch()
    / makeNoise("on.wav");
</script>

Introduction

If you include the state-machine-cat inpage script (state-machine-cat-inpage.min.js), it will extend html with three script types:

typewhat's it do?
text/x-smcat accepts smcat
text/x-scxml accepts SCXML
text/x-smcat-json accepts smcat's abstract syntax tree format

Put a script referencing it in the header of your HTML document with a defer attribute, or put it at the bottom of the document. Example for the first: 

<script 
  src="https://state-machine-cat.js.org/state-machine-cat-inpage.min.js"
  defer>
</script>

The script attribute takes a few optional data attributes you can use to influence how things get rendered:

attribute description
data-direction Direction the graph should be rendered in. Possible values: top-down (default), left-right, bottom-top, right-left.
data-output-type What you want to have rendered. Defaults to svg, but other values include smcat, scxml and json (to get the AST),
data-engine As long as state-machine-cat uses GraphViz it'll possible to pass the engine GraphViz uses. Defaults to dot; other values are e.g. fdp and osage
data-desugar Boolean attribute that, when included, will make the render engine desugar the diagram.

Examples

Read from an url

Pass the url in the src attribute of the script tag:

<script type="text/x-smcat" src="samples/cat.smcat"></script>

In this example the states get layed out from left to right instead of top down because the script includes the data-direction attribute:

Orientation with data-direction

<script 
  type="text/x-smcat"
  src="samples/cassetteplayer.smcat"
  data-direction="left-right">
</script>

De-sugaring a state-machine

Not a legal or practical state machine, but most of state machine cat's features are in.

as is

<script 
  src="samples/desugarable.smcat" 
  type="text/x-smcat" 
  data-direction="left-right" 
</script>

de-sugared

<script 
  src="samples/desugarable.smcat" 
  type="text/x-smcat" 
  data-direction="left-right" 
  data-desugar=true>
</script>

Using SCXML

It's possible to embed SCXML it or use a src attribute that points to a URL, just like it is works for smcat. 

<script type="text/x-scxml">
  <scxml xmlns="http://www.w3.org/2005/07/scxml" version="1.0">
    <state id="off">
      <onentry><log expr="winston.log('switched lamp on')"/></onentry>
      <transition event="switch_flipped" target="on"/>
    </state>
    <state id="on">
      <transition event="switch_flipped" target="off"/>
    </state>
  </scxml>
</script>

or likewise, using a src attribute to load of from somewhere else: 

<script type="text/x-scxml" src="samples/sprint-states.scxml"></script>

Output formats other than svg

If you, for some reason don't want to render an svg, but instead a different format, or even the source itself (formatted), you can specify an output type. Here e.g. to render an smcat source into its abstract syntax tree (AST): 

<script type="text/x-smcat" data-output-type="json">
  off: entry/ <log expr="winston.log('switched lamp on')"/>,
  on;

  off => on: switch_flipped;
  on => off: switch_flipped;
</script>
